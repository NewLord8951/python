## 🗓 Этапы разработки по методологии Waterfall
 ### 1. 📌 Сбор требований
    - **Функциональные требования**:
        - Базовые операции (+, -, *, /)
        - Дополнительные функции (%, √, память (M+, M-))
    - **Нефункциональные требования**:
        - Платформа: Веб
        - Интерфейс: минималистичный дизайн
 ### 2. 🎨 Проектирование
    - **Архитектура**:
        - Модуль `core.js` — логика вычислений
        - Модуль `ui.js` — обработка кликов и отображение
    - **Дизайн**:
        - Макет в Figma
 ### 3. 💻 Реализация
    - **Технологии**: JavaScript, React, HTML, CSS
    - **Этапы разработки**:
        1. Разработка ядра (функции вычислений)
        2. Создание интерфейса (HTML/CSS)
        3. Интеграция логики и UI
 ### 4. 🧪 Тестирование
    - **Юнит-тесты** (Jest) для функций
    - **UI-тесты** (клики, отображение)
    - **Сценарии тестирования**:
        - `2 + 2 = 4`
        - `5 / 0 = "Ошибка"`
 ### 5. 🚀 Внедрение
    - **Деплой**: GitHub Pages / Netlify
    - **Документация**: README с инструкцией
 ### 6. 📞 Поддержка
    - Исправление багов
    - Добавление новых функций (инженерный режим)
